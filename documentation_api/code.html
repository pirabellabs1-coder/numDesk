Guide d'intégration API

Apprenez à intégrer les capacités vocales de Vocalia dans votre application. Ce guide couvre les bases, de l'authentification à l'écoute des événements en temps réel.

key

Authentication

L'API Vocalia utilise des clés API pour authentifier les requêtes. Vous pouvez visualiser et gérer vos clés API dans le tableau de bord Vocalia.

Vos clés API portent des privilèges importants, assurez-vous donc de les garder en sécurité ! Ne partagez pas vos clés API secrètes dans des zones accessibles au public.

Pro Tip

Toutes les requêtes API doivent être effectuées via HTTPS.

cURL content_copy 
curl -X GET https://api.vocalia.ai/v1/me \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
phone_forwarded

Making your first call

Pour initier un appel sortant, envoyez une requête POST à l'endpoint /calls. Vous devez spécifier le numéro de destination et l'identifiant de l'agent IA à utiliser.

  • check_circle to: Le numéro de téléphone au format E.164.
  • check_circle agent_id: L'ID de votre configuration d'agent Sonic.
  • check_circle webhook: (Optionnel) URL pour recevoir les mises à jour.
JavaScript (SDK) content_copy 
const vocalia = require('@vocalia/sdk')('sk_test_...');

await vocalia.calls.create({
  to: '+33612345678',
  agent_id: 'agent_8f92j0',
  prompt_params: {
    user_name: 'Alice'
  }
});

Webhook events

Les webhooks permettent à Vocalia d'envoyer des notifications en temps réel sur les événements de vos appels. Configurez votre URL de destination dans le panneau de configuration.

call.completed

Déclenché lorsqu'un appel se termine avec succès.
